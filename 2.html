SQL 인젝션 공격

  • 오랜 기간 계속해서 OWASP TOP 10의 가장 심각한 리스크로 선정된 인젝션 리스크의 대표적인 공격.
  • 웹사이트의 회원정보 등 개인정보를 탈취하고자 오래전부터 지속적으로 사용되어온 공격이다.

    • 잘 알려진 사례로 2011년 소니의 데이터베이스가 SQL 인젝션 공격을 당해 약 100만 명의 회원정보와 350만 개의 디지털 쿠폰 등의 공격자에게 유출되었다. 2015년에는 국내의 온라인 커뮤니티 사이트인 뽐뿌 사이트에서 SQL 인젝션 공격으로 190만 명의 회원정보가 유출되었다. 또 유명한 해킹 그룹인 어나니머스가 WTO 세계무역기구에 속한 사이트를 SQL 인젝션 공격으로 공격하여 수천 명의 재직자 정보를 유출한 사례도 있다.

    SQL 인젝션 취약점은 웹 서버 영역의 데이터베이스로 전송되는 SQL 쿼리문을 사용자가 임의로 조작할 수 있는 경우 발생한다. 공격자는 이 취약점을 이용해 데이터베이스에 저장되어 있는 다른 사용자의 개인정보 등 허가되지 않은 정보에 접근하거나 데이터를 변조할 수 있다.

    SQL 인젝션 공격 대응

  • SQL 인젝션 공격과 같은 모든 파라미터 입력값 조작으로 이루어지는 공격은 입력값 검증을 통해 대응할 수 있다. 특히 화이트리스트 검증을 사용하는 것이 더욱 좋다.
    • SQL 인젝션 공격은 시큐어 코딩을 통해 좀 더 구체적으로 대응할 수 있다. 목적은 사용자가 입력한 값은 SQL 쿼리문에서 오직 데이터로만 사용되어야 하지 SQL 쿼리문의 구조에 영향을 줄 수 없도록 해야 한다.
